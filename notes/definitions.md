What is Docker Desktop?
Docker Desktop is software that helps you run applications inside containers on your computer.
Think of it like this - 
* A Virtual Machine (VM) runs a full operating system.
* A Docker Container runs only the application and its required files.
Because of this, containers are - 
* Faster
* Lightweight
* Use less RAM and storage
* Start quickly

What Docker Desktop Does ?
1. Container Engine -
Runs Docker containers on your laptop or desktop system.
2. Easy Management UI - 
   Provides a graphical interface to:
* Start/Stop containers
* View logs
* Monitor CPU/RAM usage
* Open container terminals
* Manage images and networks
3. Networking Between Containers - 
Docker allows multiple containers/services to communicate with each other.
Example:
* Wazuh Manager
* Elasticsearch/OpenSearch
* Dashboard
* Database
all communicate internally.
4. Simplifies Installation -
Instead of:
* Downloading packages manually
* Fixing dependencies
* Configuring services individually
* Docker downloads ready-made images and runs them automatically.

What is a Container?
A container is a small isolated environment that runs an application along with everything it needs.
It contains -
* Application code
* Libraries
* Dependencies
* Runtime environment
* Configuration/settings
This ensures the application works the same way on any computer.

What is Wazuh?
Wazuh is an open-source security monitoring platform used for -
* Threat detection
* Log analysis
* File integrity monitoring
* Vulnerability detection
* Security event monitoring
* Incident response
It is commonly used in - 
* SOC (Security Operations Center)
* Blue Teaming
* Threat Hunting
* Security Labs

What is Wazuh Cloud?
Wazuh Cloud is the cloud-hosted version of Wazuh. It allows you to -
* Monitor endpoints remotely
* Collect and analyze security logs
* Detect threats and suspicious activity
* Access a web-based security dashboard
* Build a SOC lab without hosting your own server
Because it is cloud-hosted -
* No need to maintain infrastructure
* Easier setup
* Faster deployment

What are wazuh components?
The Wazuh platform consists of several key components main is Wazuh Agent, Wazuh Server, Wazuh Indexer & Wazuh Dashboard

What is Wazuh Dashboard?
The Wazuh Dashboard is the web interface used to -
* View security alerts
* Monitor endpoint activity
* Search logs
* Create visualizations
* Investigate incidents
* Analyze threats
* SOC analysts use the dashboard to monitor systems in real time.

What is a Wazuh Agent?
A Wazuh Agent is software installed on endpoint devices such as (Windows systems, Linux systems, Servers, Virtual Machines)
The agent collects and sends security data to the Wazuh server/dashboard. The agent can send -
* Logs
* Login activity
* File changes
* Security events
* Malware alerts
* Process activity
* System information

What is wazuh server ?
Wazuh Server is the brain of Wazuh. It receives logs from agents, checks them against security rules, and creates alerts when it finds suspicious activities.

What is Wazuh Indexer ? 
Wazuh Indexer is the storage component of Wazuh. It stores all logs, alerts, and security data collected from monitored systems. It helps users quickly search and view security events in the Wazuh Dashboard while ensuring data is stored safely and efficiently. / Wazuh Indexer is responsible for storing, indexing, and managing security logs and alerts, enabling fast searching and visualization of data in the Wazuh Dashboard.

What is a Log?
A log is a record of events generated by a system, application, or device. 
Logs help track (User activity, Errors, Warnings, Security events, System behavior)
Logs usually contain -
* Timestamp
* Event description
* Severity level
* Source
* User/process information
Logs are essential for -
* Troubleshooting
* Threat detection
* Incident response
* Digital forensics

What are the type of logs?
Types of Logs
-> System Logs - Operating system events. Example (Service started, User login, Driver failure)
-> Application Logs - Events generated by applications. Example (Browser errors, Database failures)
-> Security Logs - Security-related activities. Example (Failed login attempts, Privilege escalation, Malware detection)
-> Network Logs - Network communication events. Example (Firewall logs, DNS queries, VPN connections)

What is Sysmon?
Sysmon (System Monitor) is a Microsoft Sysinternals tool that records detailed Windows system activity.
It logs (Process creation, Network connections, File creation, Registry modifications, PowerShell activity, Driver loading, Malware behavior)
Sysmon logs are extremely valuable for SOC analysts and threat hunters.

Why SOC Analysts Use Sysmon ?
SOC teams use Sysmon for deeper visibility into endpoint activity.
Traditional antivirus solutions may miss suspicious behavior, but Sysmon provides detailed telemetry for detecting -
* Malware execution
* Persistence mechanisms
* Suspicious PowerShell commands
* Lateral movement
* Command-and-control communication
* Credential dumping
* Privilege escalation
By analyzing Sysmon logs, analysts can identify malicious or abnormal activity more effectively.

What is SIEM?
SIEM stands for Security Information and Event Management
A SIEM platform -
* Collects logs from multiple devices
* Analyzes security events
* Detects threats
* Generates alerts
* Helps analysts investigate incidents
Examples (Wazuh, Splunk, QRadar, Microsoft Sentinel)

What is SOC?
Security Operations Center (SOC) is a team responsible for monitoring and defending systems against cyber threats.
SOC analysts -
* Monitor alerts
* Investigate incidents
* Analyze logs
* Respond to attacks
* Hunt threats

What is Threat Hunting?
Threat Hunting is the proactive process of searching for hidden threats or suspicious activity inside systems and networks.
Instead of waiting for alerts, analysts actively investigate abnormal behavior.

What is Endpoint Monitoring?
Endpoint monitoring means continuously tracking activities on devices such as (Laptops, Servers, Virtual machines, Workstations)
The goal is to detect suspicious or malicious activity.

What is File Integrity Monitoring (FIM)?
File Integrity Monitoring tracks changes to important files.
It detects (File modifications, File deletions, Unauthorized changes, Malware tampering)
Wazuh uses FIM to monitor sensitive files and directories.

What is an Alert?
An alert is a warning generated when suspicious or important activity is detected.
Example alerts (Multiple failed logins, Malware detected, Suspicious PowerShell execution, File integrity changes)
Alerts help SOC analysts investigate potential threats.

What is Threat Detection?
Threat detection is the process of identifying malicious or suspicious activity within systems or networks.
This can include (Malware, Unauthorized access, Data exfiltration, Ransomware, Exploitation attempts)

What is Telemetry?
Telemetry refers to the continuous collection of system and security data from endpoints.
Examples (Process execution, Network connections, Login activity, File changes)
Sysmon and Wazuh agents generate telemetry data for analysis.
