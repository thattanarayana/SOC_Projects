                                                         #LOGGLY PROJECT
			                                      -:LOGGLY Setup Process/Steps:-

1. Create Loggly Account
-> Sign up on Loggly website = https://www.loggly.com/signup/
-> Navigate to "Add a Log Source"

2. Configure Log Source in Loggly dashboard 
-> Select Linux System Log
-> Choose Manual Configuration

3. Update System & Install rsyslog service/demon
-> sudo apt update
-> sudo apt install rsyslog -y

4. Verify Installation & check the status of the service
-> Check rsyslog version "rsyslogd -v"
-> Check service status "systemctl status rsyslog"

5. Configure rsyslog for Loggly
-> Create configuration file "vim /etc/rsyslog.d/22-loggly.conf"
-> Add the following configuration (compatible with rsyslog version > 7)
-> After the update of given configuraction save exit the vim press "Esc, :wq "

# Setup disk assisted queues
$WorkDirectory /var/spool/rsyslog
$ActionQueueFileName fwdRule1
$ActionQueueMaxDiskSpace 1g
$ActionQueueSaveOnShutdown on
$ActionQueueType LinkedList
$ActionResumeRetryCount -1

# Loggly Template (REPLACE TAG)
template(name="LogglyFormat" type="string"
 string="<%pri%>%protocol-version% %timestamp:::date-rfc3339% %HOSTNAME% %app-name% %procid% %msgid% [3ec56256-f62c-44a6-b242-cfef35ba03df@41058 tag=\"kali-linux\"] %msg%\n")

# Send logs to Loggly
*.* action(type="omfwd" protocol="tcp" target="logs-01.loggly.com" port="514" template="LogglyFormat")

6. Restart rsyslog Service & check the staus of rsyslog service
-> Restart rsyslog service "sudo systemctl restart rsyslog"
-> Check status rsyslog service "sudo systemctl status rsyslog"

7. Verify Logs in Loggly Dashboard
-> Open Loggly Dashboard
-> Check incoming logs under Live Tail / Search
