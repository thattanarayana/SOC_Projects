					                                  -: Unauthorized File/Folder Access Detection using Wazuh SIEM :- 

Project Overview - 
-> This project demonstrates how to detect unauthorized file access, file modifications, and file deletion activities on a Windows endpoint using Wazuh SIEM and File    Integrity Monitoring (FIM).
-> The lab simulates real-world SOC monitoring scenarios where sensitive files are continuously monitored for suspicious or unauthorized changes.
-> Through this project, security analysts can understand how to -
* Monitor sensitive files in real time
* Detect unauthorized file activity
* Configure Windows auditing
* Investigate alerts in Wazuh SIEM
* Perform SOC alert analysis and triage

Project Objective - 
The primary objective of this project is to monitor and detect -
* Unauthorized file access
* File modifications
* File deletions
* Suspicious activity on sensitive folders
* using Wazuh File Integrity Monitoring (FIM), Windows Security Auditing & Wazuh Cloud SIEM

Step 1 — Create Sensitive Folder
	 Create a folder "C:\SensitiveData" in Windows system for monitoring.
	 Create sample files (salary.txt, passwords.txt, finance.xlsx) (echo new content >> testing.txt)
Step 2 — Enable Windows Audit Policy
	 Opene PowerShell as Administrator run below cmd to enabled file system auditing.
 	 cmd - auditpol /set /subcategory:"File System" /success:enable /failure:enable
 	 cmd to verfiy - auditpol /get /subcategory:"File System"
	 Output should be - Success and Failure
Step 3 — Configure Wazuh File Integrity Monitoring (FIM)
	 Open (C:\Program Files (x86)\ossec-agent\ossec.conf) as admin update the below configuraction
	 <directories realtime="yes">C:\SensitiveData</directories> in side <syscheck> 
Step 4 — Configure Security Event Collection
	 Open (C:\Program Files (x86)\ossec-agent\ossec.conf) as admin removie below event id's
	 EventID 4663, 4660, 4656
Step 5 — Restart Wazuh Agent using below cmd's in powershell as admin 
	 NET STOP WazuhSvc
	 NET START WazuhSvc
Step 6 — Configure Folder Auditing
 	 Configure auditing permissions of (C:\SensitiveData) in windows VM
	 Navigate -> Properties → Security → Advanced → Auditing
	 Added auditing entry:
	 Setting			Value
	 Principal			Everyone
	 Type				Success
	 Applies To			This folder, subfolders and files
	 Permissions		Full control
Step 7 — Generate File Activity
	 Perform following activities like cretae, edit, delete files
Step 8 — Monitor Alerts in Wazuh
	 Use the following search query (rule.groups:syscheck) in Wazuh Dashboard to find the analysis
	 Detecte alerts - Integrity checksum changed, File modified, File deleted, File monitoring events

Investigation Workflow - 
During alert investigation, the following analysis was performed -
* Identified which file was modified
* Verified who performed the activity
* Checked timestamps of file events
* Validated whether activity was authorized or suspicious
* Correlated Windows Security Events with Wazuh alerts
